## 팩토리 패턴 Factory Pattern
#### ; 객체를 사용하는 코드에서 객체 생성 부분을 떼어내 추상화한 패턴.
#### ; 상속 관계에 있는 두 클래스에서 상위 클래스가 중요한 뼈대를 결정하고, 하위 클래스에서 객체 생성에 관한 구체적인 내용을 결정하는 패턴.

<br />

### 특징
 1) 상위 클래스와 하위 클래스가 분리됨 > 느슨한 결합도
 2) 상위 클래스에서는 인스턴스 생성 방식에 대해 알 필요가 없음 > 더 많은 유연성
 3) 객체 생성 로직이 따로 분리되어 있음 > 코드를 리팩터링해도 한 곳만 고칠 수 있어 유지 보수성 증가됨
